# Project Overview: Telegram Mini App Booking System

## 📋 Описание проекта

Система бронирования услуг малого бизнеса на базе Telegram Mini App. Предназначена для быстрой записи клиентов на услуги.

### Бизнес-модель
- **НЕ SaaS** — каждая копия = отдельный бизнес
- **Один деплой = один клиент**
- **Отдельная база данных** для каждого клиента
- **Отдельный Telegram бот** для каждого клиента
- Индивидуальная конфигурация для каждого бизнеса

### Целевые бизнесы
- Барбершопы
- Салоны красоты
- Массажные салоны
- Автомойки
- Студии маникюра
- Косметологические кабинеты

---

## 🎯 Основные цели

### 1. Скорость записи
Весь процесс от открытия бота до подтверждения записи должен занимать **≤ 2 минуты**.

### 2. Простота использования
- Минимум текста
- Максимум визуальных подсказок
- Интуитивная навигация
- Нативный для Telegram UX

### 3. Гибкость конфигурации
Владелец бизнеса должен легко настраивать:
- Название и контакты
- График работы
- Список услуг с ценами и длительностью
- Выходные дни

---

## 👥 Пользовательские роли

### Клиент (End User)
- Просматривает услуги
- Выбирает дату и время
- Оформляет запись
- Получает уведомления

### Администратор (Business Owner)
- Настраивает бизнес-профиль
- Управляет услугами
- Видит расписание
- Получает уведомления о записях

---

## 🔄 Основной пользовательский сценарий

```
1. Пользователь открывает Telegram бота
   ↓
2. Бот отправляет кнопку "Записаться"
   ↓
3. Открывается Mini App (SPA)
   ↓
4. Пользователь видит список услуг
   ↓
5. Выбирает услугу → открывается календарь
   ↓
6. Выбирает дату → показываются слоты
   ↓
7. Выбирает время → открывается форма
   ↓
8. Вводит имя и телефон
   ↓
9. Подтверждает запись
   ↓
10. Получает уведомление в боте
```

## 📱 Технологический стек

### Frontend (Mini App)
- **React 18** — UI библиотека
- **TailwindCSS** — стилизация
- **React Router** — навигация (SPA)
- **Telegram Mini App SDK** — интеграция с Telegram
- **Zustand** — state management
- **date-fns** — работа с датами
- **Vite** — сборка

### Backend API
- **Node.js 18+** — runtime
- **Express** — web framework
- **PostgreSQL** — база данных
- **Prisma** — ORM
- **node-cron** — scheduled jobs

### Bot
- **Node.js** — runtime
- **Telegraf** — Telegram Bot framework

### Database
- **PostgreSQL** — основная БД
- **Prisma** — ORM и миграции

### Deployment
- **Mini App**: Vercel / Cloudflare Pages
- **Backend**: VPS / Docker / Railway / Render
- **Database**: Supabase / Railway / Neon

## 🚀 Принципы разработки

### 1. Mobile-First
Приложение создаётся для мобильных устройств. Desktop — вторичен.

### 2. Native Feel
Интерфейс должен ощущаться как нативная часть Telegram.

### 3. Minimalism
Минимум элементов, максимум ясности.

### 4. Speed
Быстрая загрузка, мгновенный отклик.

### 5. Accessibility
Понятный интерфейс для всех пользователей.

---

## ⚠️ Критические ограничения

1. **Не использовать SSR** — только SPA
2. **Не перегружать экраны** — максимум 3-4 элемента
3. **Не требовать регистрацию** — запись без аккаунта
4. **Не собирать лишние данные** — только имя и телефон
